# 复新生涯平台建设项目基本情况 (Project Brief)

## 项目概要 / Overview

复旦大学新闻学院在学生就业与职业发展方面已有坚实基础，但生涯引导服务仍存在一些难点：
- 就业信息发布渠道主要依赖微信群、公众号等传统方式
- 信息来源分散，岗位分类粗放，无法形成标准化、可搜索的数据库

学院拟开发面向学生的就业服务智能体和职业发展智能体平台：
- 实现多源信息的自动采集、标准化处理和语义化分类
- 为学生提供精准岗位推荐、简历优化和职业路径分析
- 整合学生职业能力、实习经历和兴趣偏好，形成可视化职业发展画像

## 项目需求 / Requirements

### 1. 就业信息的聚合发布与反馈收集
- 从微信群、招聘网站、微信公众号获取岗位信息
- OCR 识别、文本清洗、结构化处理
- 按岗位类型、发布时间、截止日期智能分类
- 管理员手动维护岗位、编辑「学院本周推荐岗位」
- 后台编辑 / 格式化表格导入创建岗位
- 用户基于关键词搜索 + 多级筛选（岗位、城市、行业）
- 投递方式：跳转官网 OR 平台内简历投递

### 2. 简历优化与人岗匹配
- 个人中心上传简历
- 浏览岗位时，基于岗位需求 + 用户简历提供对话式简历优化与岗位匹配建议 *(v1 暂不实现)*
- 通过意向城市 / 岗位类型 / 学历等硬性条件初筛
- 结合简历与岗位描述的语义相似度，进行个性化推荐

## 团队分工 / Team

### 前端 / Frontend

**用户端**
- 首页：岗位列表、推荐岗位轮播图、搜索框与筛选栏
- 岗位详情页：公司、地点、描述、开放日期、投递方式（平台内问卷 + 简历选择器）
- 个人中心：我的资料 / 我的简历 / 我的投递 入口
- 我的资料：基本信息、求职意向维护
- 我的简历：上传、删除、预览
- 我的投递：已投递记录（支持修改投递、截止状态判断）
- ~~对话交互页：岗位匹配分析、简历优化建议~~ *(v1 暂不实现，已从前端移除)*

**管理员端**
- 岗位列表编辑页：批量下线 / 删除 / 推荐状态
- 岗位创建页：手动新增、格式化表格导入
- 岗位详情编辑页：修改岗位内容、投递方式
- 岗位简历管理页：查看简历并批量导出

### 后端 / Backend
- 用户认证：学校统一认证 (CAS / OAuth2)
- 用户信息：资料、简历上传/删除（格式与大小校验）、站内投递
- 岗位信息：增删改、搜索筛选、首页推荐管理、批量导出

### 算法 / Algorithm
- 岗位信息收集：本地微信群消息读取、公众号链接抓取、文本清洗与字段规范化
- 简历优化：识别简历文件、归一化（教育背景、项目经历、技能特长）、完整性与匹配度分析 *(v1 暂不实现)*
- 岗位推荐：硬性条件初筛 + 语义相似度计算

## 项目支持

- AI 工具账号 (Cursor / Claude / ChatGPT) 与 API Token 算力资源
- 服务器开发环境、复旦大学域名、UIS 接口
- 项目酬金 + 项目实践证书
